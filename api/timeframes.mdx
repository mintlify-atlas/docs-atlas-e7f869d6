## Overview

The XAUUSD Trading Bot analyzes market data across multiple timeframes to provide comprehensive trading signals. Each timeframe offers different perspectives on market trends and patterns.

## Supported Timeframes

The bot supports six standard timeframes, from daily to 5-minute charts:

<CodeGroup>

```python Available Timeframes
# Timeframes are returned in the market_data dictionary
result = bot.run_analysis()

for timeframe in result['market_data'].keys():
    print(timeframe)
    
# Output:
# D1
# H4
# H1
# M30
# M15
# M5
```

```python Iterate Through Timeframes
result = bot.run_analysis()

for tf, data in result['market_data'].items():
    print(f"Timeframe: {tf}")
    print(f"Data: {data}")
    print("-" * 50)
```

</CodeGroup>

## Timeframe Definitions

### D1 - Daily

<ParamField path="D1" type="string">
  **Daily timeframe** - Each candle represents one trading day
  
  **Use cases:**
  - Long-term trend identification
  - Major support/resistance levels
  - Swing trading strategies
  - Position trading entries/exits
  
  **Typical analysis period:** 30-90 days of data
</ParamField>

### H4 - 4-Hour

<ParamField path="H4" type="string">
  **4-Hour timeframe** - Each candle represents 4 hours of trading
  
  **Use cases:**
  - Medium-term trend analysis
  - Intraday swing trading
  - Identifying session-based patterns
  - Confirming daily trends
  
  **Typical analysis period:** 10-30 days of data
</ParamField>

### H1 - 1-Hour

<ParamField path="H1" type="string">
  **1-Hour timeframe** - Each candle represents 1 hour of trading
  
  **Use cases:**
  - Short-term trend analysis
  - Intraday trading opportunities
  - Session breakout strategies
  - Fine-tuning entry/exit points
  
  **Typical analysis period:** 5-15 days of data
</ParamField>

### M30 - 30-Minute

<ParamField path="M30" type="string">
  **30-Minute timeframe** - Each candle represents 30 minutes of trading
  
  **Use cases:**
  - Scalping and day trading
  - Quick trend reversals
  - News event trading
  - Precise entry timing
  
  **Typical analysis period:** 3-7 days of data
</ParamField>

### M15 - 15-Minute

<ParamField path="M15" type="string">
  **15-Minute timeframe** - Each candle represents 15 minutes of trading
  
  **Use cases:**
  - Active day trading
  - Scalping strategies
  - Quick profit targets
  - High-frequency analysis
  
  **Typical analysis period:** 1-3 days of data
</ParamField>

### M5 - 5-Minute

<ParamField path="M5" type="string">
  **5-Minute timeframe** - Each candle represents 5 minutes of trading
  
  **Use cases:**
  - Ultra-short-term scalping
  - Market microstructure analysis
  - Very tight stop-losses
  - Rapid trade execution
  
  **Typical analysis period:** 1-2 days of data
</ParamField>

## Multi-Timeframe Analysis

The bot performs analysis across all timeframes simultaneously to provide a comprehensive market view:

<Expandable title="Top-Down Analysis">
  The bot uses a top-down approach:
  
  1. **D1 (Daily)**: Identify the overall trend
  2. **H4 (4-Hour)**: Confirm trend direction and strength
  3. **H1 (1-Hour)**: Find potential reversal zones
  4. **M30 (30-Minute)**: Refine entry opportunities
  5. **M15 (15-Minute)**: Precise entry timing
  6. **M5 (5-Minute)**: Final entry confirmation
</Expandable>

<Expandable title="Confluence Trading">
  Signals are stronger when multiple timeframes align:
  
  ```python
  result = bot.run_analysis()
  
  # Check alignment across timeframes
  # Higher timeframes = stronger trend
  # Lower timeframes = better entry timing
  
  daily_trend = analyze_trend(result['market_data']['D1'])
  h4_trend = analyze_trend(result['market_data']['H4'])
  
  if daily_trend == h4_trend == "BULLISH":
      print("Strong bullish alignment - high probability setup")
  ```
</Expandable>

## Displaying Timeframe Data

The Streamlit application displays timeframe data in a grid layout:

```python
import streamlit as st

def display_market_data(data_str, timeframe):
    """Display formatted market data in an expander"""
    with st.expander(f"📊 {timeframe} Market Data", expanded=False):
        lines = data_str.split('\n')
        for line in lines:
            if line.strip():
                st.text(line)

# Display in 2-column grid
st.markdown("### 📈 Market Data by Timeframe")
cols = st.columns(2)

for idx, (tf, data) in enumerate(result['market_data'].items()):
    with cols[idx % 2]:
        display_market_data(data, tf)
```

## Timeframe Data Structure

Each timeframe's data is returned as a formatted multi-line string:

<ResponseField name="market_data[timeframe]" type="string">
  Formatted market data containing:
  
  - **Price Data**: Open, High, Low, Close (OHLC)
  - **Volume**: Trading volume for the period
  - **Technical Indicators**: Moving averages, RSI, MACD, etc.
  - **Trend Information**: Current trend direction and strength
  - **Support/Resistance**: Key price levels
</ResponseField>

## Best Practices

<Expandable title="Timeframe Selection by Trading Style">
  Choose timeframes based on your trading approach:
  
  | Trading Style | Primary Timeframe | Confirmation Timeframes |
  |--------------|-------------------|------------------------|
  | Position Trading | D1 | H4, H1 |
  | Swing Trading | H4, H1 | D1, M30 |
  | Day Trading | H1, M30 | H4, M15 |
  | Scalping | M15, M5 | M30, H1 |
</Expandable>

<Expandable title="Avoid Conflicting Signals">
  When timeframes show conflicting signals:
  
  ```python
  # Higher timeframe takes precedence
  if daily_signal == "BUY" and m5_signal == "SELL":
      # Trust the daily signal more
      # Wait for lower timeframes to align
      print("Wait for M5 to confirm daily trend")
  ```
</Expandable>

<Expandable title="Update Frequency">
  Different timeframes require different update intervals:
  
  - **D1**: Update once per day
  - **H4**: Update every 4 hours
  - **H1**: Update hourly
  - **M30**: Update every 30 minutes
  - **M15**: Update every 15 minutes
  - **M5**: Update every 5 minutes
  
  The bot's auto-refresh feature uses 30-minute intervals as a balanced approach.
</Expandable>
